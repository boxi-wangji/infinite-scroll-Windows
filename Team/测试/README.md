# 测试职责说明

## 角色定位

负责功能验收测试、会话恢复专项测试、快捷键全覆盖测试及回归测试。

---

## 测试环境要求

| 项目 | 要求 |
|------|------|
| 操作系统 | Windows 10 21H2+ 或 Windows 11 |
| Node.js | 18.x LTS 或 20.x LTS |
| 安装包测试 | 全新 Windows 环境（无 Node.js / Electron） |
| 测试 Shell | CMD 和 PowerShell 均需覆盖 |

---

## 测试模块

### 1. 终端基础功能
- 终端能正常渲染、输入、输出
- 颜色、ANSI 转义序列正常显示
- 中文字符不乱码
- 终端大小随窗口自适应调整

### 2. 无限滚动画布
- 新增行（Ctrl+Shift+↓）：行数累加，画布可向下无限滚动
- 行间滚动（Ctrl+Wheel）：流畅滚动到目标行
- 多列布局：单行内多个终端并排显示

### 3. 快捷键
- 所有快捷键见 `Team/项目经理/DECISIONS.md` D-004
- 终端处于焦点时，Ctrl+W 等快捷键不能被 shell 拦截（需测试 edge case）

### 4. 会话持久化（方式 B 专项）
- 正常关闭后重启：布局（行数、列数）恢复
- cwd 恢复：新 shell 在原目录打开
- scrollback 恢复：历史输出可见，有 `[Session restored]` 标记
- 强制终止进程后重启（任务管理器 kill）：测试是否有部分恢复

### 5. 笔记面板
- 内容输入、保存、重启后读取

### 6. 安装包测试
- NSIS 安装包在全新环境安装，无报错
- 卸载后无残留文件（除 `%APPDATA%` 配置外）

---

## 已知 Edge Cases

| 场景 | 预期行为 |
|------|----------|
| 终端进程被外部 kill（如任务管理器） | `pty:exit` 事件触发，面板显示"已断开"，不崩溃 |
| 关闭最后一个 cell | 该行消失；若所有行都关闭，保留一个空行或提示新建 |
| 字体大小超出范围（< 8 或 > 24） | 夹在边界值，不允许继续调整 |
| 窗口极小（宽度 < 200px） | FitAddon 不崩溃，最小 cols=1 rows=1 |
| 快速连续按 Ctrl+Shift+↓ 新增多行 | 不卡顿，不重复创建 |
